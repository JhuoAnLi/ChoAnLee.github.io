CH1 認識C語言

Created2022-02-14|Updated2022-02-25|C語言
|Word count:836|Reading time:3min|Post View:

Why C?👍

IEEE spectrum (IEEE綜覽) comes with the listing sheet of top🔝 programming language 2021. The list of the programming language is based on popularity.

C can help!💁🏻

1. Need to write high-performance code for a new game
2. Program an Arduino
3. use that advanced third-party library in your iPhone app
4. C is a language for small, fast programs

C works at a much lower-level than most other languages

it creates code that is a lot closer to what machines really understand 💡

🔎 Understanding C gives you a much better idea of what is really going on
🔎 C can even help you better understand other languages as well

C’s history of development

  • C was developed at AT&T Bell Laboratories in the early 1970s
  • C was derived from a language called B, hence the name C to indicate it is a successor of language B(BCPL)

    The B language cannot handle these objects of different sizes, nor does it provide a separate operator to operate on them.

The characteristics of C

高效率的編譯式語言

為 編譯式語言 ，執行速度遠比直譯式快

直譯器(interpreter) 編譯器(compiler)
一次編譯一個敘述，然後執行，佔用的記憶體較少，但執行的速度會變慢 將整個程式碼編譯成機器碼(machine code)，然後執行

中階語言

介於高階與低階之間的語言

  • C 語言兼具低階語言的優點，與高階語言的特色
    🔥低階語言如組合語言與機器碼，適合電腦閱讀
    🔥高階語言貼近人類語言習慣，如BASIC，適合人類閱讀

靈活的控制流程

可以容易的設計出具有結構化 及模組化 的程式語言

可攜性佳🏋️‍♀️

  • 想 跨平台來執行C語言 ，通常只要修改極少部分的程式碼，再重新編譯即可執行
  • C語言可以直接依 記憶體的位址來存取變數 ，以提高程式執行的效率

The way C works

1. Source:

You start off by creating a source file.The source file contains human-readable C code.

2. Compile:

You run your source code through a compiler.The compiler checks for errors,and once it’s happy,it compiles the source code.

3. Output:

The compiler creates a new file called an executable.This file contains machine code,a stream of 1s and 0s that the computer understands.And that’s the program you can run.

C語言與其他程式之間的關係

C++ 與 Java 均是以 C 為根基，再加上 OOP 技術，使得它們活躍於視窗與網路程式設計

程式規劃與實作的流程

  • 規劃程式
  • 撰寫程式碼及註解
  • 編譯程式碼
  • 執行程式
  • 除錯與測試
  • 程式碼的修飾與儲存

程式的錯誤

找出錯誤的過程稱為除錯(debug)

語意錯誤(semantic error) 語法錯誤(syntax error)
程式的執行結果不是預期結果;執行才知，如想要算1+1，輸出卻是3 程式中有不合語法的敘述，Compile時通常就會知道，如main拼成maan

原始程式編譯及連結的過程

目的檔(Object file) 函數庫(Library) 連接程式(Linker)
存目的碼的檔案 。 目的碼是由編譯器產生的代碼，通常是機器碼;產生最終可執行的程式檔的中間產物;通常完成最終檔案的生成後，會自動被刪掉 將一些常用的功能集中;方便二次使用;可透過連結程式與其他目的檔/函式庫連接組合使用 連接目的檔們 (Object files);連接函數庫們 (Libraries);產出最終的可執行檔案

pros and cons of C
